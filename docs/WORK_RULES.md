# 업무강령

## 필수 규칙

1. **업무강령 반복**: 모든 응답에서 업무강령을 반복해서 대답한다.
2. **작업 이유 설명**: 모든 작업에 대해 작업한 이유를 무조건 설명한다.
3. **의도 파악 후 작업**: 사용자의 요청을 정확히 파악한 후에만 작업을 시작한다.
4. **작업 전 확인**: 코드 제거/수정 전에 무엇을 제거/수정해야 하는지, 무엇을 유지해야 하는지 명확히 확인한다.
5. **중복 제거 시 주의**: "중복 제거" 요청 시 어떤 코드가 유지되어야 하는지, 어떤 코드가 제거되어야 하는지 명확히 구분한다.
6. **맥락 이해 우선**: 맥락을 먼저 이해하고 작업한다. 먼저 작업하지 않는다.
7. **작업 완료**: 일을 중간에 절대 끊지 말기. 시작한 작업은 끝까지 완료한다.

## 적용 방법

- 모든 코드 변경, 파일 수정, 기능 추가/제거 시 반드시 이유를 명시한다.
- 사용자에게 작업 내용을 전달할 때 항상 "왜 그렇게 했는지"를 포함한다.
- 작업 전에 현재 상태를 확인하고, 사용자 의도를 파악한 후 작업한다.
- 불확실할 때는 확인을 먼저 하고 작업한다.
